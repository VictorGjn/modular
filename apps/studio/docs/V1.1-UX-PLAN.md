# V1.1 UX Plan — Agent Builder Consolidation

> Combines: V1.0.4 post-mortem learnings + Victor's UX feedback (Mar 13) + open issues

## Design Principles

1. **Left panel = the builder.** C'est là qu'on crée l'agent (brain dump → generate). Le panel central visualise/édite le résultat.
2. **Missing Sources + Insights = les stars.** Ce sont les éléments différenciants — l'IA qui dit "il te manque ça" et "j'ai trouvé ça dans tes données".
3. **No dead buttons.** Si un bouton ne fait rien, il n'existe pas.
4. **Edit ≠ Generate.** Le panel central permet d'ajuster manuellement, pas de générer section par section.

### 1.2 Fix Tags input — special characters impossible
- **File:** `src/panels/AgentBuilder.tsx` line 498-499
- **Current:** `val.split(',').map(t => t.trim()).filter(Boolean)` — splits on every comma keystroke
- **Fix:** Debounce the split, or better: switch to a chip/pill input that commits on Enter/Tab/comma. Temporary fix: only split+commit on blur, allow free typing in the field.
- **Effort:** 30min (debounce) or 1.5h (chip input)

## Phase 2 — Missing Sources Prominence (P1, ~3h)

> The "gaps" are Modular's unique value prop. Make them unmissable.

### 2.1 Persistent Missing Sources banner
- **File:** `src/panels/SourcesPanel.tsx` ~line 200
- **Current:** Small conditional block buried after the generate button
- **New design:**
  ```
  ┌─────────────────────────────────────┐
  │ ⚠ 3 MISSING SOURCES                │
  │                                     │
  │ 🔴 Competitor pricing (ground_truth)│
  │    "No pricing data found in..."    │
  │    [+ Add source] [Dismiss]         │
  │                                     │
  │ 🔴 API documentation (evidence)    │
  │    "Referenced but not provided..." │
  │    [+ Add source] [Dismiss]         │
  └─────────────────────────────────────┘
  ```
- **Behavior:**
  - Rendered as a sticky/prominent card at the top of SourcesPanel (after brain dump, before knowledge)
  - Each gap has an action button: opens FilePicker filtered by suggested type, or suggests MCP connection
  - Badge count in the panel header tab: `Sources (3 gaps)`
  - Red accent border — impossible to miss
- **Closes:** Part of differentiation story

### 2.2 Gap resolution flow
- When user clicks "+ Add source" on a gap → open FilePicker or ConnectionPicker pre-filtered
- After adding a source that matches the gap type → auto-dismiss the gap
- Track resolved vs dismissed gaps in store

## Phase 3 — Facts/Insights Front and Center (P1, ~3h)

> Facts extracted from sources = the agent getting smarter. Show this prominently.

### 3.1 Promote Facts out of Memory tab
- **File:** `src/panels/SourcesPanel.tsx` ~line 982+
- **Current:** Facts are inside the "Memory" tab — a secondary view most users won't find
- **New:** Create a dedicated "Insights" section visible in the main SourcesPanel flow, right after Missing Sources:
  ```
  ┌─────────────────────────────────────┐
  │ 💡 INSIGHTS (12 extracted)          │
  │                                     │
  │ 🔵 "API rate limit is 100 req/min" │
  │    from: api-docs.md · evidence     │
  │                                     │
  │ 🔴 "Revenue target is $2M ARR"     │
  │    from: strategy.pdf · ground_truth│
  │                                     │
  │ 🟣 "Users prefer dark mode 3:1"    │
  │    from: survey.csv · hypothesis    │
  │                                     │
  │ [Show all 12] [+ Add manually]      │
  └─────────────────────────────────────┘
  ```
- **Show top 3-5 by relevance**, expandable to full list
- **Knowledge type color dots** (🔴🟡🔵🟢🟣) per the existing type system
- **Source attribution** — which file/source did this come from

### 3.2 Fact → Agent binding
- Facts should visually connect to the agent config — "these facts inform the agent's knowledge"
- In AgentBuilder (center panel), show a small "Based on X insights" indicator near the knowledge budget bar
- Click → scrolls to the insights section in SourcesPanel
